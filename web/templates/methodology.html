Methodology

Overview

This platform fits Gaussian Process Regression (GPR) models to experimental data. It supports two analysis modes depending on the structure of your dataset:

  • Std mode — designed for datasets with replicate measurements. Observations sharing the same condition variables are grouped, and the GP models the per-group mean or standard deviation.
  • Mean mode — designed for datasets without replicates. The GP is fit directly to each row, treating each observation independently.

Both modes can be run simultaneously (Both mode) to compare replicate-aggregated vs. raw-row modeling on the same dataset.

Data Preprocessing

Before fitting, all numerical control variables are standardized using StandardScaler (zero mean, unit variance). Categorical variables are one-hot encoded via pd.get_dummies, and the resulting binary columns are included as control variables.

In Std mode, the measurement column is also standardized before grouping. In Mean mode, the output column is standardized before fitting to improve GP optimization stability.

Optional log-transformation can be applied to selected variables prior to fitting. When a variable is log-transformed, prediction grids are generated in log space and the axis is displayed on a log scale.

Replicate Grouping (Std Mode)

Rows sharing identical values for all control variables are treated as replicates. Each group is summarized into a single row containing the per-group mean, standard deviation, and count. Groups with fewer than 2 replicates are dropped, as standard deviation is undefined.

An optional outlier filter can be applied per group: observations more than k standard deviations from the group mean are excluded before computing summary statistics.

Gaussian Process Regression

GPR is a non-parametric Bayesian method that places a prior over functions and updates it given observed data. The posterior provides both a predicted mean and a calibrated uncertainty estimate at every point.

The default kernel is ConstantKernel × RBF with ARD (Automatic Relevance Determination), using one length scale per input dimension. Hyperparameters are optimized by maximizing the log marginal likelihood, with 5 random restarts to avoid local optima.

Kernel Options

  • RBF — Radial Basis Function; assumes infinitely smooth functions.
  • Matérn — Controls smoothness via the ν parameter (1/2, 3/2, or 5/2); more appropriate for functions with limited differentiability.
  • Rational Quadratic — Equivalent to an infinite mixture of RBF kernels at different length scales; captures multi-scale variation.

Kernel Structure

  • ConstantKernel × base — scales the output variance of the base kernel.
  • base + WhiteKernel — adds an explicit iid noise term; useful when observation noise is not already encoded in the alpha parameter.

Noise (Alpha)

In Std mode, observation noise is set per data point using the per-group variance estimates:

  • Std — alpha = (standard deviation)²
  • SEM — alpha = (standard deviation / √count)²
  • Constant — alpha = 0.05 for all points

In Mean mode, a constant alpha is always used since individual replicate variance is unavailable.

Model Evaluation

Training-set performance is reported using three metrics:

  • R² — coefficient of determination; 1.0 is a perfect fit.
  • MAE — mean absolute error in standardized units.
  • RMSE — root mean squared error in standardized units.

Visualization

Plots are generated for each unique combination of categorical variable values. An All plot showing all data is always included first.

  • 1D — GP mean ± 95% confidence interval over a single input variable, with other variables held at their median.
  • 2D — Contour plots of GP predicted mean and uncertainty over two input variables, with remaining variables held at their median.

Three color schemes are available: Default, Colorblind-safe, and High Contrast.

What it does

  • Fits GPR models to experimental data through a guided 4-step workflow
  • Steps: Upload → Feature Engineering → Analysis Setup → Kernel & Training

Analysis Modes

  • Experimental Uncertainty (Std) — groups replicate rows by condition; GP models the per-group mean or std
  • Experimental Data (Mean) — fits GP directly to each row; no grouping applied
  • Both — runs both pipelines simultaneously for side-by-side comparison

How it works

  • Define new columns from existing ones using Python-style expressions
  • Reference any column by name; NumPy available as np
  • New columns are immediately available in the analysis setup step
  • Optional — can be skipped entirely

Example

rxn_concentration = concentration * volume

Numerical Variables

  • Standardized with StandardScaler — zero mean, unit variance
  • One scaler fit per column; axes inverse-transformed for display
  • Std mode: measurement column scaled before grouping
  • Mean mode: output column scaled before fitting

Categorical Variables

  • One-hot encoded via pd.get_dummies
  • Binary dummy columns appended to the control variable list; not scaled

Log Scale

  • Optional flag per variable
  • Prediction grids use log-spaced points; axis displayed on log scale

How groups are formed

  • Rows sharing identical control variable values → treated as replicates
  • Each group summarized into: mean, std, count
  • Groups with fewer than 2 replicates are dropped (std undefined)

What is GPR?

  • Non-parametric Bayesian method — places a prior over functions, updates it with data
  • Outputs a predicted mean and a calibrated uncertainty at every point
  • Uses ARD (Automatic Relevance Determination) — one length scale per input dimension
  • Hyperparameters optimized via log marginal likelihood with 5 random restarts

Kernel Options

  • RBF — infinitely smooth; good baseline for smooth physical processes
  • Matérn — ν = 1/2, 3/2, or 5/2; controls smoothness; preferred for experimental data
  • Rational Quadratic — mixture of RBFs at different scales; captures multi-scale variation

Kernel Structure

  • ConstantKernel × base — learns overall signal magnitude
  • base + WhiteKernel — adds explicit iid noise on top of per-point alpha

Noise (Alpha)

  • Std mode — Std: α = σ²  |  SEM: α = (σ/√n)²  |  Constant: α = 0.05
  • Mean mode — constant α = 0.05 (no replicate variance available)

Reported Metrics

  • R² — coefficient of determination; 1.0 = perfect fit on training data
  • MAE — mean absolute error in standardized units
  • RMSE — root mean squared error in standardized units

All metrics are computed on the standardized target values used during fitting.

General

  • One plot per unique categorical combination; All plot always included first
  • All axes inverse-transformed to original units for display

1D Plots

  • One input swept across its range; all others held at median
  • Shows predicted mean + shaded 95% CI (mean ± 1.96 × std)
  • Std mode (mean target): y-axis auto-unscaled to measurement column units
  • Std mode (std target): Unscale std axis toggle controls units

2D Plots

  • Two inputs define a prediction grid; rest held at median
  • Left panel — GP predicted mean, always in original units
  • Right panel — GP uncertainty (posterior std); toggle-controlled unscaling

Color Schemes

  • Default — tab colors + viridis
  • Colorblind-safe — Wong palette + cividis
  • High Contrast — black-anchored palette + binary
